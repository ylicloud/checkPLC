日常启动

  1. 首次：双击 setup.bat（创建虚拟环境、装依赖、可选生成语音）
  2. 每次通检：双击 run.bat，浏览器打开 http://127.0.0.1:8000

页面操作

  1. 连接：填 IP / 机架 / 槽；无 PLC 可勾选 Mock。可调语音速度后点「连接」。
  2. 配置：勾选实际 DI/DQ/AI/AQ 模块，填起始地址与点数；「保存并下发」写入 configs/配置名.json。
  3. DI：端子给 24V，看大字通道号并听语音（槽内 1…N，不累加）。
  4. AI：加电流，变化超过约 10% 才播报；回落到已测过的原值不再播，便于测下一通道。
  5. DQ：选模块，点通道强制输出，万用表测端子。
  6. AQ：PLC 自动阶梯电流（通道 n → 约 3+n mA），本页监视。

真机通检前需在 TIA 下载通检 DB/SCL（见仓库 plc/、docs/）。笔记本与 PLC 同网段，放行 TCP 102。

做什么

从 TIA Portal 工程自动读取模块组态（起始地址、点数等），生成 Web「配置」页可用的 configs/柜名.json，免手工抄地址。

TIA V20（PLC 离线）
  → export.bat 导出 .aml
  → aml_to_cabinet.py 转成 configs/*.json
  → Web「配置」页加载柜名

涉及脚本 / 工具

工具路径与作用
export.bat tools\tia-openness-export\
调用 TIA Openness CAx，把当前/指定工程硬件导出为 .aml。首次运行会 dotnet build。
aml_to_cabinet.py scripts\aml_to_cabinet.py
解析 AML，按 DI/DQ/AI/AQ 填满最多 20 槽，写出柜配置 JSON。
示例 AML tools\tia-openness-export\samples\demo_cabinet.aml
无 Portal 时可用来验证转换脚本。

前置条件（工程机）

  1. 安装 TIA Portal V20（含 Openness）
  2. Windows 用户加入组 Siemens TIA Openness 后重新登录
  3. 安装 .NET SDK（编译导出工具）
  4. 确认存在：
    C:\Program Files\Siemens\Automation\Portal V20\PublicAPI\V20\Siemens.Engineering.dll
    路径不同时设环境变量 TIA_PUBLICAPI
  5. 导出时 PLC 须离线（CAx 要求）

① 导出 AML

建议先在 Portal 中打开目标工程。

cd tools\tia-openness-export
export.bat --out D:\Temp\柜A.aml

常用参数：

REM 指定工程文件（可 --new 强制新开）
export.bat --project "D:\path\质检查线.ap20" --out D:\Temp\柜A.aml --new

REM 只导出某一站
export.bat --device "S7-1200 station" --out D:\Temp\柜A.aml

也可用 Portal：设备视图右键站 → 导出 CAx（若菜单可用）。

② 转为柜配置 JSON

在仓库根目录执行（质检笔记本只需 AML 文件 + Python 即可）：

cd D:\repos\checkPLC
python scripts\aml_to_cabinet.py D:\Temp\柜A.aml -o configs\柜A.json --name 柜A --ip 192.168.0.1

参数说明：

  • -o：输出 JSON 路径
  • --name：Web 配置名（与文件名对应）
  • --ip：PLC IP（优先；也可尝试从 AML 猜测）
  • --dry-run：只打印结果不写文件

无 Portal 时用示例验证：

python scripts\aml_to_cabinet.py tools\tia-openness-export\samples\demo_cabinet.aml -o configs\demo_from_aml.json --name demo_from_aml

③ Web 一键加载

  1. 双击 run.bat 打开通检页
  2. 「配置」页下拉选中柜名 → 加载
  3. 对照地址一览与 TIA 设备视图；确认后可「保存并下发」
  4. 「连接」页填 IP（或已由 JSON 带入）后开始通检

映射与注意

  • Digital+Input→DI，Digital+Output→DQ，Analog+Input→AI，Analog+Output→AQ
  • 每类最多 20 槽；本机集成 DI/DQ 会拆成两个逻辑槽
  • Length=0 / 无效地址官方可能不导出
  • 若解析出 0 个模块，把真实 AML 样例发回以便适配命名空间
  • 更细说明：仓库 docs/tia-openness-export.md、tools/tia-openness-export/README.md

语音说明

  • 优先播放预录整段词片（快）；未生成词片时控制台会提示，请运行生成脚本。
  • 连接页可调速度：1 / 1.2 / 1.5 / 1.8 / 2 倍。
  • 通道号按模块内 1…N 播报（如两块 32 点模块，都是 1…32）。
  • >20 简化读音：21→「二一」，32→「三二」（不读「十」）。
  • 新播报会打断旧播报，不排队。
